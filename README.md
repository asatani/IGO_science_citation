# IGO Science Citation Analysis

Analysis code for studying how scientific research is cited in Intergovernmental Organization (IGO) policy documents.

## Overview

This repository contains analysis notebooks for:
- Building master datasets of policy-cited scientific papers
- Analyzing citation distances between policy-cited and non-policy-cited papers
- Visualizing co-citation networks using ForceAtlas2
- Country-level and journal-level policy citation bias analysis
- Identifying Highly IGO-Cited Scientists (HIC-Sci)
- IGO (Intergovernmental Organization) citation network analysis
- Institution ranking by policy citation impact

## Data provenance and reproducibility

This repository contains the code for data processing and analysis. The underlying raw data from Overton and Scopus are proprietary and cannot be redistributed due to licensing restrictions.

### Overton

**1. DOI list of IGO-cited papers**

Overton data were retrieved on June 7, 2024, via the "Search scholarly articles cited in policy" interface, filtered by "Cited by policy source with organisation types = intergovernmental organizations (IGOs)" and "Published in year = 2015–2023".

Place the resulting DOI list CSV files in:
```
DATA_DIR/doi_list/
```

**2. Full policy citation records**

Full citation records—indicating which IGO cited which paper—were retrieved using the Advanced Search Query Builder in Overton with the DOIs obtained above.

Place these files in:
```
POLICY_CITE_BASE/
├── yearly_policy_cite/       # Annual DOI lists (used by notebook 01)
│   ├── cited2015/
│   │   └── *.csv
│   ├── cited2016/
│   │   └── *.csv
│   └── ...                   # One folder per year
└── detaild_data/
    └── articles-202*.csv     # Detailed Overton article records (used by notebooks 05, 06)
```


### Scopus

Scopus metadata were obtained from a licensed institutional dump and matched to the Overton DOI list by DOI.

Place Scopus files in:
```
SCOPUS_BASE/
├── paper/
│   ├── doi.pickle            # DOI string for each paper
│   ├── year.pickle           # Publication year
│   ├── authids.pickle        # List of Scopus author IDs per paper
│   ├── journal.pickle        # Scopus journal/source ID
│   ├── afids.pickle          # List of affiliation IDs per paper
│   ├── reference.pickle      # List of referenced DOIs/Scopus IDs
│   ├── num_cited.pickle      # Number of times cited (forward citations)
│   ├── title.pickle          # Paper title
│   ├── doctype.pickle        # Document type classification
│   ├── fwci_c4.pickle        # Field-Weighted Citation Impact
│   ├── countries.pickle      # Country affiliations per paper
│   └── abstract.pickle       # Paper abstracts
├── author/
│   └── author_name.pickle    # Author ID → name mapping
├── citation/
│   ├── citedby.pickle        # Reverse citation index (papers citing each paper)
│   └── reference.pickle      # Forward reference list per paper
├── affiliation/
│   ├── af_name2.pickle       # Affiliation ID → institution name mapping
│   └── af_country.pickle     # Affiliation ID → country mapping
├── clustering/                # Pre-computed; generated by notebook 01
│   └── leiden_CPM/
│       ├── partition_res_1e-07_margedto_6.pickle  # Pre-computed cluster assignments
│       └── partition_res_1e-07_info.pickle         # Cluster metadata
├── embedding/                 # Pre-computed; generated by notebook 01
│   └── abstract_openai-3-small_1536_UMAP12/
│       └── *.pickle          # Abstract embeddings (OpenAI text-embedding-3-small)
└── journal/
    ├── journal_name.pickle   # Journal ID → journal name mapping
    └── IF_1972_2023_df.pickle # Journal Impact Factor time series (1972–2023)
```


### Configuration

Specify local data paths in `config/settings.py`. For example:
```python
from pathlib import Path

DATA_DIR         = Path("data")
POLICY_CITE_BASE = Path("data/policy_citation")
SCOPUS_BASE      = Path("data/scopus")
```

### Reproducing the analysis

`01_build_master_data.ipynb` constructs the master dataset by loading the DOI list, matching it to Scopus records, merging metadata, and applying deduplication and filtering. Researchers with licensed access to Overton and Scopus can follow this workflow to reproduce the analyses. Raw or derived proprietary data are not included in this repository. The repository documents the required input files and workflow needed to reproduce the analyses for researchers with licensed access to the underlying data sources.

## Notebook Structure

| Notebook | Description |
|----------|-------------|
| `01_build_master_data.ipynb` | Data construction, UMAP embedding, Leiden clustering, and time-series analysis |
| `02_distance_analysis.ipynb` | Calculate citation distance from policy-cited papers |
| `03_network_visualization.ipynb` | ForceAtlas2 network visualization (GPU-accelerated with cuGraph) |
| `04_country_analysis.ipynb` | Country-level and journal-level policy citation bias analysis |
| `05_author_analysis.ipynb` | Author metrics (Gini, NAHHI, Top 30%), HIC-Sci identification, Rich-Club analysis |
| `06_igo_analysis.ipynb` | IGO citation metrics and inter-IGO network analysis |
| `07_institution_ranking.ipynb` | Affiliation-level rankings and KNN similarity analysis |
| `08_HIC-Sci_table1_code.ipynb` | Keyword analysis for Highly IGO-Cited Scientists |

## Requirements

### Python Dependencies

Install dependencies with:
```bash
pip install -r requirements.txt
```

GPU-accelerated libraries (`cuml`, `cugraph`) are required for notebooks 02 and 03, and should be installed via conda/RAPIDS (see [RAPIDS installation guide](https://docs.rapids.ai/install)).

The `OPENAI_API_KEY` environment variable must be set to use the OpenAI API for abstract embeddings (notebook 01).

### Data Dependencies
- Scopus database exports (paper metadata, citations, affiliations, journals)
- Policy citation data from Overton/Altmetric
